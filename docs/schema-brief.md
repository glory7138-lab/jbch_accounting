# 회계 프로그램 스키마 브리핑

## 1. 분석 대상 엑셀

- `월말결산_SAMPLE.xlsx`
  - 시트: `결산양식`, `참여현황 및 주요관리항목 지출`, `주간보고자료`, `회계결산 표지`, `Sheet1`
  - 특징: 월말 결산서, 주차별 수입/지출, 계정과목별 요약, 인원/금액 집계
- `헌금현황_SAMPLE.xlsx`
  - 시트: `주별 헌금 장부기록 1-7`, `2026년 헌금 전체 누계`, `인원집계`, `헌금 봉투 번호 (2026)`, `입금전표 (수요집금)`, `입금전표`
  - 특징: 일자/월 기반 헌금 흐름, 성도 번호/이름 기반 기준정보, 입금전표 양식
- `회계장부_SAMPLE2.xlsx`
  - 시트: `통합계정`, `일반계정`, 다수의 회계/헌금 계정 시트, `계정코드`, `작성상의 주의사항`
  - 특징: 거래 원장 형식, 회계코드 마스터, 계정별 잔액 추적

## 2. 실제로 파악된 핵심 컬럼 구조

### 월말결산
- `계정과목`, `수입(전월/당월)`, `지출(전월/당월)`, `비고`
- 주간 보고는 `1주차~5주차`, `합계`, `차이` 형태

### 헌금현황
- `날짜/월`, `성도 번호`, `이름`, `회/부서`, `헌금 종류`, `항목별 금액`, `전체 합계`
- 입금헌금표는 `입금일자`, `헌금 항목`, `금액`, `비고` 계열

### 회계장부
- 공통 원장 컬럼
  - `월`, `주차`, `입금/지출 관련 번호`, `회계코드`, `관리항목`, `적요`, `수입금액`, `지출금액`, `잔액`, `비고`
- 계정코드 마스터 컬럼
  - `회계코드`, `대분류관리항목`, `중분류관리항목`, `세부관리항목(보고용)`, `차변계정`, `대변계정`

## 3. 권장 DB 스키마

### `funds`
회계 단위. 예: 통합계정, 일반계정, 후원회비, 선교헌금 등.

| 컬럼 | 설명 |
|---|---|
| id | PK |
| code | 내부 코드 |
| name | 회계/기금 이름 |
| is_active | 사용 여부 |

### `accounts`
회계코드 마스터.

| 컬럼 | 설명 |
|---|---|
| id | PK |
| code | 회계코드 |
| name | 대표 계정명 |
| major_category | 대분류 관리항목 |
| middle_category | 중분류 관리항목 |
| report_category | 보고용 세부항목 |
| debit_account | 차변계정 힌트 |
| credit_account | 대변계정 힌트 |
| normal_side | 기본 방향 |

### `members`
성도/개인 기준정보.

| 컬럼 | 설명 |
|---|---|
| id | PK |
| member_no | 성도 번호 |
| name | 이름 |
| department_name | 회/부서/구역 |
| gender_or_section | 성별/구분 |
| age_or_class | 나이/반 |
| source_sheet | 원본 시트 |

### `accounting_periods`
월 마감 관리.

| 컬럼 | 설명 |
|---|---|
| id | PK |
| year | 회계연도 |
| month | 회계월 |
| status | open/closed |
| closed_at | 마감일시 |

### `vouchers`
전표 헤더.

| 컬럼 | 설명 |
|---|---|
| id | PK |
| voucher_no | 전표번호 |
| voucher_date | 거래일자 |
| entry_type | income/expense |
| description | 적요 |
| amount | 대표 금액 |
| fund_id | 회계/기금 FK |
| account_id | 대표 계정 FK |
| member_id | 관련 성도 FK |
| counterparty | 상대방 |
| note | 비고 |
| source_workbook/source_sheet | 원본 추적 |

### `voucher_lines`
분개 상세.

| 컬럼 | 설명 |
|---|---|
| id | PK |
| voucher_id | 전표 FK |
| line_no | 순번 |
| account_id | 계정 FK |
| fund_id | 회계/기금 FK |
| debit | 차변 금액 |
| credit | 대변 금액 |
| description | 상세 적요 |
| note | 비고 |

### `import_batches`
엑셀 분석/가져오기 이력.

### `ai_suggestion_logs`
적요 기반 AI 계정추천 로그.

## 4. 설계 의견

- 엑셀은 보고서 형식과 입력 원장이 섞여 있어서, **보고서 테이블을 그대로 DB로 옮기기보다 전표 중심 정규화**가 맞음.
- 월말결산/주간보고는 `vouchers`를 집계해서 다시 생성하는 구조가 유지보수에 유리함.
- `계정코드`, `기금`, `성도`는 마스터 테이블로 분리하는 게 확장성 측면에서 가장 좋음.
- AI 추천은 `accounts` 마스터를 근거로 동작하도록 설계해야 실제 업무 흐름에 잘 맞음.
